# Работа с документацией

## Общие положения

Документация создается на языке [Markdown](https://daringfireball.net/projects/markdown/).

## Файловая структура

Файлы документации должны находиться в директории `local/docs/`.

Каждому разделу документации должен соответствовать один файл.

Каждый файл документации должен содержать заголовок первого уровня с названием раздела.

Подразделы каждого раздела размещаются в поддиректории с названием раздела.

Названия файлов и директорий должны содержать только маленькие буквы латинского алфавита, цифры и знак дефиса.

Список разделов документации в виде многоуровневого нумерованного списка со ссылками на файлы находится в файле `local/docs/index.md`. Название раздела в списке должно совпадать с названием раздела (заголовком первого уровня) в файле документации.

## Форматирование

Для обозначения ненумерованных списков должен использоваться символ `*`.

Для обозначения нумерованных списков должены использоваться символы `1.`. Каждый элемент списка должен нумероваться цифрой `1.` (Markdown сам расставляет номера элементов, так проще писать).

При необходимости использовать знак тире, пишем `--`. Представление Markdown на Bitbucket не поддерживает длинные тире `---`, поэтому они использоваться не должны.
